## 第 1 步：准备 PostgreSQL 数据库

如果你本机已有 PostgreSQL，创建库：

```sql
CREATE DATABASE smart_logistics;
```

然后执行建表脚本：

```bash
psql -U postgres -d smart_logistics -f sky-server/src/main/resources/sql/backend1_business_init.sql
```

执行后检查：

```sql
SELECT * FROM users;
SELECT * FROM vehicles;
SELECT * FROM cargo;
SELECT * FROM cargo_vehicle_binding;
SELECT * FROM cargo_status_logs;
```

你应该看到 5 个演示账号、1 辆车、1 个货物、1 条绑定关系。

## 第 2 步：让 Spring Boot 连接 PostgreSQL

我已经给你加了 PostgreSQL 驱动，并把默认 `dev` 配置改成智慧物流本地开发配置：

```txt
sky-server/src/main/resources/application-dev.yml
```

启动：

```bash
mvn spring-boot:run -pl sky-server
```

默认连接 `localhost:5432/smart_logistics`，用户名 `postgres`。如果你的数据库有密码，启动前设置：

```bash
export DB_PASSWORD=你的数据库密码
```

## 第 3 步：先不要写所有接口，先写 users 登录

目标接口：

```txt
POST /api/v1/auth/login
GET  /api/v1/users/me
```

你要做的分层：

```txt
AuthController
  -> AuthService
  -> UserMapper
  -> users 表
```

正式基建已经放好，结构是：

```txt
entity/LogisticsUser.java
dto/LoginDTO.java
vo/LoginVO.java
vo/LoginUserVO.java
mapper/LogisticsUserMapper.java
resources/mapper/LogisticsUserMapper.xml
service/AuthService.java
service/impl/AuthServiceImpl.java
controller/AuthController.java
```

练习顺序：

1. `LogisticsUserMapper` 负责查 `users` 表。
2. `AuthServiceImpl` 负责校验 username/password。
3. 输入密码会先做 MD5，再和 `users.password_hash` 比较。
4. 登录成功后生成 JWT accessToken / refreshToken。
5. Controller 只负责接收请求和返回 `ApiResponse<LoginVO>`。

如果你之前已经执行过旧 SQL，运行这个脚本把密码改成 MD5：

```bash
psql -h localhost -p 5432 -U postgres -d smart_logistics \
  -f sky-server/src/main/resources/sql/backend1_update_users_password_md5.sql
```

测试：

```bash
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"dispatcher","password":"123456"}'
```

## 第 4 步：再写车辆列表

目标接口：

```txt
GET /api/v1/vehicles
GET /api/v1/vehicles/{plate}
POST /api/v1/vehicles
```

分层：

```txt
VehicleController
  -> VehicleService
  -> VehicleMapper
  -> vehicles 表
```

先做最简单版本：

1. `GET /vehicles` 返回全部车辆。
2. 再加 keyword/status 筛选。
3. `GET /vehicles/{plate}` 用主键查。
4. `POST /vehicles` 新增时自动生成 `vinTopic = plate.replace("·", "-")`。

## 第 5 步：再写货物列表和绑定

目标接口：

```txt
GET  /api/v1/cargo
GET  /api/v1/cargo/{cargoId}
POST /api/v1/cargo
POST /api/v1/cargo/bind
POST /api/v1/cargo/unbind
PUT  /api/v1/cargo/{cargoId}/status
GET  /api/v1/cargo/{cargoId}/status-logs
```

绑定时必须检查：

1. 车辆存在。
2. 货物存在。
3. 车辆没有 ACTIVE 绑定。
4. 货物没有 ACTIVE 绑定。
5. 插入 `cargo_vehicle_binding`。

状态上报时必须做两件事：

1. 更新 `cargo.status`。
2. 插入 `cargo_status_logs`。

## 第 6 步：给后端 2/3 提供查询方法

这些方法先放到 `BusinessLookupService`：

```java
getVehicleByPlate(String plate)
getVehicleByVinTopic(String vinTopic)
getVehicleByImei(String imei)
getCargoById(String cargoId)
getActiveCargoByPlate(String plate)
getActiveBindingByPlate(String plate)
getActiveBindingByCargoId(String cargoId)
isVehicleAvailable(String plate)
```

后端 2 做 MQTT GPS 时会用 `vinTopic / imei` 找车辆和货物。
后端 3 做告警和调度时会用 `plate / cargoId` 查当前任务。

## 第 7 步：替换 starter 的内存数据

现在临时数据在：

```txt
sky-server/src/main/java/com/sky/logistics/service/impl/LogisticsStarterServiceImpl.java
```

你不要一次删光。按顺序替换：

1. 先替换 `login()`。
2. 再替换 `vehicles()` 和 `vehicleDetail()`。
3. 再替换 `cargo()` 和 `cargoDetail()`。
4. 最后替换 `bindCargo()`、`updateCargoStatus()`。

每替换一个方法，就用 curl 测一次。
