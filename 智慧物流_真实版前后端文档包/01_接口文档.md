# 智慧萌宠护送 IoT 平台接口文档

> 对应代码：`pet-safe-transit/sky-take-out/sky-server/`
> 核对时间：2026-07-13
> Base URL：`http://localhost:8080/api/v1`
> Swagger/Knife4j：`http://localhost:8080/doc.html`
> WebSocket：`ws://localhost:8080/api/v1/ws?token=<JWT>`
> MQTT Broker：`mqtt://localhost:1883`，如做 TLS 则使用 `mqtts://localhost:8883`
> 数据格式：JSON / UTF-8
> 时间格式：ISO 8601 UTC，例如 `2026-06-29T06:15:00Z`
> 坐标：WGS-84，统一字段 `lat`、`lng`

---

## 0. 版本约束

| 项 | 约定 |
|---|---|
| API 前缀 | `/api/v1` |
| 认证方式 | `Authorization: Bearer <JWT_TOKEN>` |
| 分页页码 | `page` 从 1 开始 |
| 成功 code | `0` |
| 通用错误 | `40001/40101/40301/40401/40901/50001` |
| 车牌展示格式 | `沪A·C0291` |
| MQTT Topic 车牌格式 | `沪A-C0291`，将 `·` 替换为 `-` |

### 0.1 业务命名对齐

当前 Java 类、数据库表和 URL 为兼容原智慧物流代码，继续使用 `Cargo`、`cargo` 和 `/cargo`。在萌宠护送产品中，一条 `cargo` 记录表示“一次萌宠护送运单”，不是普通货物。

| 接口字段/实体 | 产品含义 |
|---|---|
| `cargoId` | 萌宠护送运单号 |
| `cargoType` | 宠物类型或运输对象描述 |
| `cargo_vehicle_binding` | 萌宠运单与护送车辆绑定 |
| `cargo_status_logs` | 护送节点状态日志 |
| `cargo-events` | 车厢/设备事件，保留协议名称 |

### 0.2 Swagger/Knife4j 调试规则

1. 打开 `http://localhost:8080/doc.html`。
2. 页面应只显示 `智慧物流接口`。
3. 先调用 `POST /api/v1/auth/login`，账号可用 `dispatcher / 123456`。
4. 复制返回的 `data.accessToken`。
5. 调需要登录的接口时，在 `Authorization` 请求头填写 `Bearer <accessToken>`。

如果接口文档页面打不开，先确认后端已启动、Maven 已重新加载，并且 `application.yml` 中启用了 `spring.mvc.pathmatch.matching-strategy=ant_path_matcher`。如果仍看到 C 端/员工/分类接口，说明后端没有重启或浏览器缓存了旧页面。

### 0.3 认证与权限约束

除下列公开接口外，`/api/v1/**` 和兼容路径 `/api/**` 都由 `LogisticsAuthInterceptor` 校验 JWT：

```txt
POST /auth/login
POST /auth/face-login
POST /auth/refresh
GET  /files/{dateDir}/{fileName}
OPTIONS 预检请求
```

当前代码中的关键角色限制：

| 能力 | 允许角色 |
|---|---|
| 用户列表、新增用户、删除用户 | `ADMIN` |
| 用户详情、人脸状态及绑定管理 | 本人或 `ADMIN` |
| 普通用户修改资料 | 仅本人，且不能修改 `role` |
| 新增/修改车辆 | `WAREHOUSE` / `DISPATCHER` / `ADMIN` |
| 删除车辆、下发调度指令 | `DISPATCHER` / `ADMIN` |
| 新增/修改/绑定/解绑运单 | `WAREHOUSE` / `DISPATCHER` / `ADMIN` |
| 取消运单 | `SHIPPER` / `DISPATCHER` / `ADMIN` |
| 更新运输状态 | `WAREHOUSE` / `DRIVER` / `DISPATCHER` / `ADMIN` |
| 查看和处理告警 | `WAREHOUSE` / `DISPATCHER` / `ADMIN` |
| 通过/拒绝改址 | `DISPATCHER` / `ADMIN` |

注意：当前后端已实现角色级鉴权，但运单所有者、司机任务归属等行级数据过滤尚未完整落地。调用方不能把“前端看不到菜单”等同于已经完成资源隔离。

```json
  "password": "123456",
  "name": "新调度员",
  "role": "DISPATCHER",
  "phone": "13800000006"

```

**Response data**


### 2.8 修改用户

**权限**：本人或 `ADMIN`。非管理员请求中的 `role` 会被拒绝，不能通过修改个人资料提升权限。

```http
PUT /users/{id}
Authorization: Bearer <TOKEN>
```

**Request**

```json
{
  "name": "王调度",
  "role": "DISPATCHER",
  "phone": "13800000002",
  "password": "123456"
}
```

说明：`password` 不传则不修改密码。

### 2.9 删除用户

**权限**：仅 `ADMIN`。

```http
DELETE /users/{id}
Authorization: Bearer <TOKEN>
```

示例：

```http

```

成功时返回：

```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

## 4. 萌宠护送运单（Cargo）

### 4.1 创建萌宠运单

```http
POST /cargo
```

**权限**：`WAREHOUSE` / `DISPATCHER` / `ADMIN`。

```json
{
  "cargoId": "PET-20260713-001",
  "cargoType": "金毛犬·布丁",
  "weight": 12.5,
  "origin": { "name": "上海宠物服务中心", "lat": 31.2304, "lng": 121.4737 },
  "destination": { "name": "杭州余杭交接点", "lat": 30.2741, "lng": 120.1551 }
}
```

说明：当前 DTO 只有 `cargoType`、`weight` 和起终点等通用字段；品种、年龄、健康提示、环境阈值等专属宠物档案字段尚未进入该接口。

### 4.2 获取萌宠运单列表

```http
GET /cargo?status=IN_TRANSIT&keyword=PET&page=1&size=20
```

**权限**：已登录用户。目标设计要求 `SHIPPER` 只能返回本人运单，`DRIVER` 只能返回分配任务；当前行级过滤尚未完整实现。


### 4.4 修改萌宠运单

**权限**：`WAREHOUSE` / `DISPATCHER` / `ADMIN`。

```http

```

```json
{
  "cargoType": "金毛犬·布丁",
  "weight": 12.80,
  "origin": { "name": "上海宠物服务中心", "lat": 31.2304, "lng": 121.4737 },
  "destination": { "name": "杭州余杭交接点", "lat": 30.2741, "lng": 120.1551 }
```

说明：已送达或已取消的运单不能修改。

### 4.5 绑定运单与车辆

**权限**：`WAREHOUSE` / `DISPATCHER` / `ADMIN`。

```http
POST /cargo/bind
```

```json
{
  "cargoId": "PET-20260713-001",
  "vehicleId": 1
}
```

说明：

- 支持一车多单：同一辆车可以绑定多个 `ACTIVE` 萌宠运单。
- 一个运单同一时间只能绑定一辆车，重复绑定会返回业务错误。
- 绑定成功后，运单状态更新为 `LOADED`。

### 4.6 解绑运单与车辆

**权限**：`WAREHOUSE` / `DISPATCHER` / `ADMIN`。

```http
POST /cargo/unbind
```

```json
{
  "cargoId": "PET-20260713-001"
}
```

### 4.7 取消萌宠运单

**权限**：`SHIPPER` / `DISPATCHER` / `ADMIN`。当前 `SHIPPER` 的本人运单校验仍需补齐。

```http
POST /cargo/{cargoId}/cancel
```

说明：

- 已送达运单不能取消。
- 取消运单会自动解除当前 active 绑定。
- 取消会写入 `cargo_status_logs`。

## 12. WebSocket 实时推送

```js
const ws = new WebSocket("ws://localhost:8080/api/v1/ws?token=<JWT>")
```
