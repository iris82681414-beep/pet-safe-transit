## 1. 本次目标

本次主要参考 `高德APIMD` 文件夹中的高德 API 对接文档，把项目从“前端有页面、部分数据模拟”的状态，推进到“后端提供真实接口、前端调用真实接口、云端可运行”的状态。

重点不是一次性重写项目，而是在原有智慧物流代码风格上继续扩展：

- 后端继续使用 Spring Boot + MyBatis + XML SQL。
- DTO、VO、Entity、Mapper、Service、Controller 分层保持清晰。
- 前端继续使用现有 Vue + Vite + services/api.ts 的调用方式。
- 配置尽量使用环境变量，避免把真实密钥写死进 GitHub。
- 云端用 systemd 管理后端，用 nginx 托管前端静态资源并代理后端接口。

实现接口：

- `POST /api/v1/orders/{orderId}/address-change-impact`
- `POST /api/v1/orders/{orderId}/address-change-requests`
- `GET /api/v1/orders/{orderId}/address-change-history`
- `GET /api/v1/address-change-requests/{requestId}`
- `POST /api/v1/address-change-requests/{requestId}/approve`
- `POST /api/v1/address-change-requests/{requestId}/reject`
- `POST /api/v1/address-change-requests/{requestId}/driver-confirm`

业务逻辑：

- 先计算新地址对原路线的影响。
- 根据新增距离、预计延误、是否超出服务范围，判断影响等级。
- 低影响改址可以自动通过。
- 中高影响改址进入调度员审核。
- 改址通过后可以重新规划路线，并向司机下发新路线命令。
- 自动通过的申请不能再被人工拒绝，避免状态流转混乱。

## 5. 配置改动

修改文件：

- `sky-take-out/sky-server/src/main/resources/application-dev.yml`

新增配置项：

```yaml
spring:
  kafka:
    listener:
      auto-startup: ${KAFKA_ENABLED:true}

mqtt:
  enabled: ${MQTT_ENABLED:true}

amap:
  web-key: ${AMAP_WEB_KEY:}
  base-url: ${AMAP_BASE_URL:https://restapi.amap.com}
  connect-timeout-ms: ${AMAP_CONNECT_TIMEOUT_MS:3000}
  read-timeout-ms: ${AMAP_READ_TIMEOUT_MS:5000}
  cache-minutes: ${AMAP_CACHE_MINUTES:30}
```

密钥处理：

- GitHub 代码中不保存真实 key。
- ECS 上通过 `/etc/smartgps/backend.env` 保存真实 key。
- systemd 服务通过 `EnvironmentFile=-/etc/smartgps/backend.env` 加载。

原因：

- GitHub 有 Push Protection，会阻止包含 DeepSeek 等密钥的提交。
- 真实生产或云端环境应该通过环境变量注入密钥。
- 本地开发和云端部署可以用同一套代码，不需要改代码里的 key。

## 8. 需要注意的地方

### 8.1 GitHub 不保存真实 key

这次 GitHub 曾经拦截过一次推送，因为提交里含有真实 DeepSeek API Key。后来已经改成：

```yaml
api-key: ${DEEPSEEK_API_KEY:}
api-key: ${SILICONFLOW_API_KEY:}
web-key: ${AMAP_WEB_KEY:}
```

真实 key 只放在 ECS：

```text
/etc/smartgps/backend.env
```

不要把这个文件提交到 Git。

### 8.2 ECS 上保留了旧 docker-compose 配置

部署时 ECS 工作区的 `docker-compose.yml` 有服务器本地改动。为了不丢配置，我做了两份保护：

- `git stash` 中保存了一份：`stash@{0}: deploy-preserve-docker-compose`
- 文件备份：`docker-compose.yml.bak-before-990b69b-deploy`

后续如果需要恢复服务器原来的 compose 配置，可以从这两个地方找。

### 8.3 货车路线能力需要高德账号支持

目前普通驾车路线已经返回 `AMAP`。  
货车路线接口保留了后端扩展入口，但高德货车路线属于更高权限能力，账号没开通时会自动降级处理。

### 8.4 司机评分有业务限制

订单未签收时提交司机评分会返回：

```text
订单未签收，不能评价司机
```

这是正常业务保护，不是接口错误。
