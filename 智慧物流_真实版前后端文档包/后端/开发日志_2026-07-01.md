## 2. 货物绑定与解绑

### 2.1 一车多货规则修正

根据业务重新确认货物和车辆关系：

- 一个货物同一时间只能绑定一辆车。
- 一辆车同一时间可以绑定多个货物。

因此调整了 `cargo_vehicle_binding` 的约束：

- 保留 `ux_active_cargo_binding`，保证一个货物只有一个 active 绑定。
- 去掉车辆 active 唯一限制，改为普通索引 `idx_active_vehicle_binding`。

这样可以支持真实物流场景中的一车多货。

### 2.2 货物解绑接口

完成接口：

```txt
POST /api/v1/cargo/unbind
```

实现内容：

- 根据 `cargoId` 查询当前 active 绑定关系。
- 如果没有绑定关系，返回业务错误。
- 将绑定记录状态从 `ACTIVE` 更新为 `UNBOUND`。
- 写入 `unbound_at` 时间。
- 返回最新货物详情。

涉及文件：

```txt

CargoService
CargoServiceImpl
LogisticsCargoMapper
LogisticsCargoMapper.xml
CargoController
```

### 4.2 GPS 轨迹关联货物

修改 `GpsKafkaConsumer`，将 GPS 数据写入 TimescaleDB 时关联当前绑定货物。

```java
String cargoId = null;

- 如果车辆绑定多个货物，同一个 GPS 点为每个货物写入一条轨迹。

```

对应 SQL：

```sql

FROM cargo_vehicle_binding
WHERE vehicle_id = #{vehicleId}
  AND status = 'ACTIVE'
ORDER BY bound_at DESC
```

## 5. TimescaleDB 时间问题修正

### 5.1 入库时间固定问题

问题：

测试 MQTT 时，`gps_points.time` 一直显示固定时间，例如：

```txt
2026-07-01 02:00:00.000000 +00:00
```

原因：

- MQTT Payload 中反复发送了固定 `ts`。
- 后端原来优先使用 `gps.getTs()` 作为入库时间。

处理：

将 `GpsKafkaConsumer` 中 GPS 入库时间改成服务端接收时间：

```java
Instant ts = Instant.now();
```

这样每次发送 MQTT，TimescaleDB 中的轨迹时间都会变化。

### 5.2 东八区显示问题

问题：

TimescaleDB 查询结果显示：

```txt
2026-07-01 07:55:04.537000 +00:00
```

实际当前北京时间是：

```txt
2026-07-01 15:55:04
```

说明：

- `+00:00` 是 UTC。
- `07:55 +00:00` 等于北京时间 `15:55 +08:00`。

处理：

在 TimescaleDB JDBC URL 中加入东八区配置：

```txt
?options=-c%20TimeZone=Asia/Shanghai
```

并对本地 `gps` 数据库执行：

```sql
ALTER DATABASE gps SET timezone TO 'Asia/Shanghai';
```

验证结果：

```txt
2026-07-01 15:55:04.537+08
```

## 9. 遇到的问题和解决方式

### 问题 1：MQTTX 无法在 Wayland 下启动

现象：

```txt
Failed to connect to Wayland display
segmentation fault
```

处理建议：

使用 X11 模式启动：

```bash
ELECTRON_OZONE_PLATFORM_HINT=x11 mqttx
```

或者直接使用 EMQX / 命令行发布 MQTT 消息。

### 问题 2：MQTT 收到没有 topic 的消息

现象：

```txt
收到没有 mqtt_topic 的消息，丢弃
```

处理：

兼容不同 Spring Integration MQTT 版本中的 Topic Header：

- `mqtt_receivedTopic`
- `mqtt_topic`

保证后端能正确识别 MQTT Topic。

### 问题 3：Kafka 镜像 tag 不存在

现象：

```txt
bitnami/kafka:3.7 not found
```

处理：

改用可用 Kafka 镜像版本，并通过 `kafka-init` 自动创建 topic。

### 问题 4：TimescaleDB 查询显示 UTC

处理：

- JDBC URL 增加 `TimeZone=Asia/Shanghai`。
- 数据库默认时区设置为 `Asia/Shanghai`。
- IDEA / DataGrip 需要断开数据库连接后重新连接。

### 问题 5：后端 2 合并冲突

冲突文件：

```txt
CargoController.java
GpsKafkaConsumer.java
application-dev.yml
```

处理：

- `CargoController` 同时保留状态日志接口和追踪接口。
- `GpsKafkaConsumer` 保留一车多货写入逻辑，并接入 WebSocket 广播。
- `application-dev.yml` 保留 `smart_logistics` 和 TimescaleDB 东八区配置。
