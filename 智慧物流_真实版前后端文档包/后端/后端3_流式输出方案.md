# 流式输出方案

## 现状
前端发 `POST /assistant/chat` → 后端走完完整 RAG 链路 → 一次性返回 JSON，等 3-10 秒看到回答。

## 目标
类似 ChatGPT：AI 逐字逐 token 输出，前端实时渲染。

## 涉及文件及状态

### SSE 事件格式

后端推的 SSE 事件：

```
event: token
data: 偏

event: token
data: 航

event: token
data: 告警

... (更多 token)

event: meta
data: {"sessionId":"SESSION-xxx","sources":[...],"context":{...},"answeredAt":"2026-07-04T..."}
```

### 前端改动伪代码

```typescript
// api.ts — 新增
async function chatStream(question: string, sessionId: string, onToken: (t: string) => void, onMeta: (m: any) => void) {
  const response = await fetch('/api/v1/assistant/chat/stream', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${token}` },
    body: JSON.stringify({ question, sessionId }),
  })
  const reader = response.body!.getReader()
  const decoder = new TextDecoder()
  let buffer = ''
  while (true) {
    const { done, value } = await reader.read()
    if (done) break
    buffer += decoder.decode(value, { stream: true })
    // 按 \n\n 拆 SSE event，解析 event: token / event: meta
    // 调 onToken / onMeta
  }
}

// AssistantView.vue — send() 改为
const result = await assistantApi.chatStream(question, sessionId,
  (token) => {
    // 追加文字到最后一条 assistant 消息
    messages.value[messages.value.length - 1].content += token
  },
  (meta) => {
    sessionId.value = meta.sessionId
    messages.value[messages.value.length - 1].sources = meta.sources
    messages.value[messages.value.length - 1].time = new Date(meta.answeredAt).toLocaleString('zh-CN')
  }
)
```

### 核心设计决策

| 问题 | 决策 | 原因 |
|---|---|---|
| 流式端点独立还是替换原端点 | 新增 `/assistant/chat/stream`，原端点保留 | 向后兼容，前端不改也不影响现有功能 |
| sources/context 何时返回 | meta event，在所有 token 之后 | LLM 开始生成时 sources 已确定，但先推 token 用户体验更好 |
| 并发控制 | 后端 `new Thread()` 每请求一个线程 | 低并发场景够用，生产改为线程池 |
| LLM 流式调用方式 | `HttpURLConnection`（原生 JDK） | 无需加 WebFlux/OkHttp 依赖 |
| 错误处理 | `emitter.completeWithError(error)` | 前端可捕获 error event 显示提示 |
