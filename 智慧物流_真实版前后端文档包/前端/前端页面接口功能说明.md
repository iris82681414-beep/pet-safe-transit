## 2. 运行配置

配置集中在 `frontend/src/services/config.ts`。

| 环境变量 | 说明 | 示例 |
|---|---|---|
| `VITE_DATA_MODE` | 数据模式：`mock`、`api`、`auto` | `api` |
| `VITE_API_BASE_URL` | 后端 HTTP API 根地址 | `http://localhost:8080/api/v1` |
| `VITE_WS_URL` | WebSocket 地址 | `ws://localhost:8080/api/v1/ws` |
| `VITE_MQTT_URL` | MQTT over WebSocket 地址 | `ws://localhost:8083/mqtt` |
| `VITE_AMAP_KEY` | 前端高德地图 JS Key | 高德 Web 端 Key |
| `VITE_AMAP_SECURITY_CODE` | 高德安全密钥 | 高德安全密钥 |
| `VITE_API_PROXY_TARGET` | Vite 本地代理目标 | `http://localhost:8080` |

说明：

- `VITE_DATA_MODE=mock`：使用本地 mock 数据，不请求后端。
- `VITE_DATA_MODE=api`：使用真实后端接口。
- `VITE_DATA_MODE=auto`：如果没有配置 `VITE_API_BASE_URL`，自动走 mock；配置了则走真实接口。
- 前端接口统一从 `serviceConfig.apiBaseUrl` 开始拼接。
- 后端返回格式默认是：

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": "2026-07-06T00:00:00Z",
  "requestId": "xxx"
}
```

前端 `apiClient` 会自动取出 `data`。

## 3. 页面总览

| 路由 | 页面组件 | 页面名称 | 可访问角色 | 主要职责 |
|---|---|---|---|---|
| `/login` | `App.vue` 内置登录页 | 登录页 | 公开 | 登录、选择演示角色、进入平台 |
| `/overview` | `OverviewView.vue` | 运营总览 | 全部角色 | 总览运营指标、车辆分布、告警概览、七日趋势 |
| `/tracking` | `TrackingView.vue` | 货物追踪 | 货主、调度员、管理员 | 货物分组、轨迹地图、ETA、改址、确认收货、司机评价 |
| `/dispatch` | `DispatchView.vue` | 车辆调度 | 调度员、管理员 | 车辆筛选、路线规划、下发调度指令、联系司机 |
| `/capacity` | `CargoCapacityView.vue` | 装载管理 | 仓库管理员、调度员、管理员 | 车辆容量、货物重量、装载预估、绑定解绑 |
| `/personnel` | `PersonnelView.vue` | 人员管理 | 调度员、管理员 | 查看司机人数、司机详情、车辆详情 |
| `/driver` | `DriverView.vue` | 司机任务 | 司机、调度员、管理员 | 司机上报货物状态、执行调度指令 |
| `/alerts` | `AlertsView.vue` | 告警中心 | 仓库管理员、调度员、管理员 | 告警筛选、详情、确认、关闭、删除 |
| `/warehouse` | `WarehouseView.vue` | 仓储管理 | 仓库管理员、管理员 | 新增车辆、编辑车辆、新建货物、绑定解绑、设备在线 |
| `/assistant` | `AssistantView.vue` | 智能问答 | 货主、调度员、管理员 | 物流问答、流式输出、Markdown 显示、会话管理 |

注意：

- `src/router.ts` 里路由组件是一个空组件，真正页面由 `App.vue` 根据当前路由名动态渲染。
- 页面菜单由 `App.vue` 的 `menus` 数组控制，不同角色只显示自己有权限的菜单。
- 当前权限是前端显示层控制，后端权限校验暂时还没有作为上线级功能完成。

## 4. 公共状态与数据流

公共状态集中在 `frontend/src/stores/logistics.ts`。

| 状态 | 说明 | 主要来源 |
|---|---|---|
| `vehicles` | 车辆列表、司机、位置、状态、绑定货物 | `GET /vehicles` |
| `cargo` | 货物/运单列表、起终点、状态、绑定车辆 | `GET /cargo` |
| `alerts` | 告警列表 | `GET /alerts` |
| `devices` | 设备在线状态 | `GET /devices/status` |
| `commands` | 调度指令记录 | `POST /vehicles/{plate}/command` / WebSocket 回执 |
| `notifications` | 前端通知中心数据 | WebSocket / 本地生成 |
| `user` | 当前登录用户 | `POST /auth/login` / `GET /users/me` |

初始化流程：

1. 登录成功后保存 token 和用户信息。
2. 调用 `initialize()`。
3. 并发加载车辆、货物、告警、设备状态。
4. 建立 WebSocket 连接。
5. 订阅车辆位置、告警、调度指令回执。
6. 如果 WebSocket 断开，前端每 15 秒轮询车辆和告警数据作为兜底。

## 5. 页面功能与接口

### 5.1 登录页

代码位置：`frontend/src/App.vue`

负责功能：

- 输入账号密码登录。
- 选择演示角色。
- 登录后进入工作台。
- 退出登录。
- token 失效后自动回到登录页。
- 上传头像到后端文件服务。

使用接口：

| 动作 | 接口 | 说明 |
|---|---|---|
| 登录 | `POST /auth/login` | 返回 accessToken、refreshToken、user |
| 获取当前用户 | `GET /users/me` | 页面刷新后恢复用户 |
| 刷新 token | `POST /auth/refresh` | token 快过期时自动刷新 |
| 退出登录 | `POST /auth/logout` | 后端退出登录 |
| 上传头像 | `POST /files/upload` | 头像上传到后端文件服务 |

### 5.3 货物追踪页

代码位置：`frontend/src/views/TrackingView.vue`

负责功能：

- 按状态把货物分成三组：
  - 未开始运输
  - 运输中
  - 已完成
- 选择货物后查看当前车辆、司机、起点、终点、实时位置。
- 地图展示货物轨迹、规划路线、纠偏路线。
- 刷新实时位置、ETA、轨迹、运输时间线。
- 确认收货，完成运单。
- 联系司机，弹出司机手机号。
- 调用高德地址能力申请改址。
- 计算改址影响：额外距离、延误、费用、是否需要审核。
- 确认卸货点。
- 对已完成运单评价司机。
- 做轨迹纠偏和一键纠偏下发。

使用接口：

| 功能 | 接口 | 说明 |
|---|---|---|
| 货物列表 | `GET /cargo` | 从 Store 加载，页面分组展示 |
| 货物实时位置 | `GET /cargo/{cargoId}/position` | 获取当前坐标、速度等 |
| 货物轨迹 | `GET /cargo/{cargoId}/trajectory` | 地图轨迹线 |
| ETA | `GET /cargo/{cargoId}/eta` | 预计到达、剩余距离 |
| 运输时间线 | `GET /cargo/{cargoId}/timeline` | 装货、运输、异常、送达等事件 |
| 更新货物状态 | `PUT /cargo/{cargoId}/status` | 确认收货时改成 `DELIVERED` |
| 路线规划 | `POST /routes/truck-plan` | 起点到终点货车路线 |
| 偏航检测 | `POST /routes/deviation/check` | 判断当前车辆是否偏离规划路线 |
| 轨迹纠偏 | `POST /trajectory/correct` | 修正 GPS 轨迹点 |
| 纠偏建议 | `POST /routes/reroute-suggestion` | 生成当前点到目的地的新路线 |
| 地址提示 | `GET /amap/input-tips` | 输入关键词联想地址 |
| 地址转坐标 | `POST /amap/geocode` | 新地址解析经纬度 |
| 改址影响计算 | `POST /orders/{orderId}/address-change-impact` | 计算额外距离、延误、费用 |
| 提交改址申请 | `POST /orders/{orderId}/address-change-requests` | 生成改址申请 |
| 卸货地址建议 | `GET /orders/{orderId}/unload-address-suggestions` | 后端推荐卸货点 |
| 确认卸货地址 | `POST /orders/{orderId}/unload-address/confirm` | 保存最终卸货地址 |
| 获取司机评价 | `GET /orders/{orderId}/driver-rating` | 查看是否已评价 |
| 提交司机评价 | `POST /orders/{orderId}/driver-rating` | 完成运单后评价司机 |

### 5.4 车辆调度页

代码位置：`frontend/src/views/DispatchView.vue`

负责功能：

- 按车辆状态筛选：全部、行驶中、待命、离线。
- 查看车辆实时位置、速度、司机、电话、绑定运单。
- 点击车辆后加载车辆详情和当前任务。
- 根据车辆当前位置和货物目的地规划路线。
- 下发新路线到车载终端。
- 下发普通调度指令。
- 联系司机，弹出司机电话。
- 上传车辆图片到后端。

使用接口：

| 功能 | 接口 | 说明 |
|---|---|---|
| 车辆列表 | `GET /vehicles` | 从 Store 加载 |
| 车辆详情 | `GET /vehicles/plate/{plate}` | 点击车辆后拉取详情 |
| 车辆当前任务 | `GET /vehicles/{plate}/active-task` | 查看绑定运单和任务 |
| 货车路线规划 | `POST /routes/truck-plan` | 生成车辆到目的地路线 |
| 下发调度指令 | `POST /vehicles/{plate}/command` | 包括路线调整、普通指令 |
| 上传车辆图片 | `POST /files/upload` | 上传到后端文件服务 |
| 实时位置 | WebSocket `vehicle.position` | 刷新车辆点位 |
| 指令回执 | WebSocket `command.ack` | 更新指令状态 |

### 5.5 装载管理页

代码位置：`frontend/src/views/CargoCapacityView.vue`

负责功能：

- 选择车辆，查看车辆载重容量。
- 查看当前车辆已经绑定的货物。
- 选择待装货物，估算加入当前车辆后的载重占用。
- 判断货物是否适合加入当前车辆。
- 将货物加入当前车辆计划。
- 将货物从车辆计划中移出。
- 从装载页面跳转到仓储新建货物或车辆调度页面。

使用接口：

| 功能 | 接口 | 说明 |
|---|---|---|
| 车辆列表 | `GET /vehicles` | 从 Store 加载车辆容量、司机、状态 |
| 货物列表 | `GET /cargo` | 从 Store 加载货物、重量、绑定关系 |
| 绑定货物车辆 | `POST /cargo/bind` | 把选中货物加入当前车辆 |
| 解除绑定 | `POST /cargo/unbind` | 从车辆计划中移出货物 |

说明：

- 当前后端货物字段里没有独立 `volume` 字段，页面里的体积是根据货物类型和重量做前端估算，用于调度辅助展示。
- 后续如果后端新增 `volume`、`length`、`width`、`height`，该页面可以直接替换估算逻辑。

### 5.8 告警中心页

代码位置：`frontend/src/views/AlertsView.vue`

负责功能：

- 告警列表筛选：状态、类型、车辆。
- 查看告警详情。
- 刷新告警统计。
- 确认告警。
- 关闭告警并填写解决方案。
- 删除当前工作台里的告警条目。

使用接口：

| 功能 | 接口 | 说明 |
|---|---|---|
| 告警列表 | `GET /alerts` | 从 Store 加载 |

| 关闭告警 | `POST /alerts/{alertId}/resolve` | 状态变为已关闭 |

说明：

- 删除告警目前是前端工作台移除，不是后端物理删除。
- 确认和关闭是后端真实状态流转。

### 6.2 车辆接口 `vehicleApi`

| 方法 | 接口 | 用途 |

## 7. 实时通信

### 7.1 WebSocket

封装文件：`frontend/src/services/websocket.ts`

连接地址：

```txt
VITE_WS_URL?token={accessToken}
```

当前订阅频道：

| 频道 | 数据 | 用途 |
|---|---|---|
| `vehicle.position` | `plate`、`lat`、`lng`、`speed`、`heading`、`timestamp` | 实时刷新车辆位置 |
| `alert.triggered` | `alertId`、`alertType`、`severity`、`vehiclePlate`、`title`、`triggeredAt` | 新告警推送 |
| `command.ack` | `commandId`、`plate`、`status`、`timestamp` | 调度指令回执 |

订阅消息格式：

```json
{
  "action": "SUBSCRIBE",
  "channel": "vehicle.position",
  "plates": ["*"]
}
```

服务端推送格式：

```json
{
  "channel": "vehicle.position",
  "eventId": "EVT-001",
  "data": {
    "plate": "沪A-C0291",
    "lat": 31.2304,
    "lng": 121.4737,
    "speed": 62,
    "heading": 90,
    "timestamp": "2026-07-06T10:00:00+08:00"
  }
}
```

### 7.2 MQTT

封装文件：`frontend/src/services/mqtt.ts`

用途：

- 浏览器端 MQTT 主要用于设备联调观察。
- 正式业务页面优先使用 HTTP API + WebSocket。

主题约定：

| 主题 | 说明 |
|---|---|
| `vehicle/{vinTopic}/gps` | GPS 上报 |
| `vehicle/{vinTopic}/heartbeat` | 心跳上报 |
| `vehicle/{vinTopic}/#` | 订阅某辆车所有设备消息 |

车牌转主题：

```ts
toVinTopic('沪A·C0291') // 沪A-C0291
```

## 10. 当前前端分工建议

| 模块 | 适合负责人 | 原因 |
|---|---|---|
| 货物追踪 | 前端 1 | 和车辆位置、轨迹、ETA、改址流程关系最密 |
| 车辆调度 | 前端 1 | 和路线规划、指令下发、地图交互关系最密 |
| 登录/告警/仓储/问答 | 前端 2 | 页面数量多，但接口封装较清晰 |
| 人员管理 | 前端 2 或公共 | 主要读取 Store 数据，逻辑相对轻 |
| 新视觉原型落地 | 共同 | 需要统一设计风格后逐页替换 |
