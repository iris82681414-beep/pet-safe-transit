## 4. 高德 API 使用原则

1. **前端使用高德 JS API**：地图展示、点选地址、画路线、画车辆点。
2. **后端使用高德 Web 服务 API**：地理编码、逆地理编码、路线规划、货车规划、轨迹纠偏、POI 搜索。
3. **Web 服务 Key 不放前端**，只放后端环境变量。
4. 高德经纬度格式是 `lng,lat`，你们系统 DTO 统一字段为：

```json
{
  "lng": 120.1551,
  "lat": 30.2741
}
```

## 5. 最终演示链路

```txt
1. 客户登录，左侧没有“告警中心”
2. 客户进入订单详情，点击“申请修改收货地址”
3. 输入地址时调用高德输入提示
4. 选择地图点后，后端调用高德地理编码/路线规划
5. 系统返回：额外距离、预计延误、是否超区、额外费用、审核等级
6. 客户提交改址申请
7. 调度员审核改址
8. 审核通过后系统重新规划路线，并下发给司机
9. 调度端地图显示新路线
10. 客户订单详情显示改址进度
11. 订单签收后，客户给司机评分
12. 调度端可以查看司机评分汇总
```
