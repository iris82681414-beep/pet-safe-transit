# 前端 2：地址修改、司机评分、告警权限整改与管理页

> 你的定位：客户端和管理端表单负责人。  
> 你负责客户改址申请、司机评分、客户隐藏告警中心、调度审核页面、卸货点确认和图片上传。

```txt

8. 卸货地址确认页面
9. 客户司机评分弹窗
10. 司机评分汇总展示
11. 头像/车辆图片/异常图片上传
12. 登录页演示角色块删除
13. 运营总览页两列火车改为保留一列
```

---

## 2. 你不负责什么

```txt
1. 不负责地图上的路线绘制，交给前端 1

3. 不负责后端权限逻辑
4. 不负责大模型后端调用
```

## 3. 客户权限整改

### 3.1 菜单隐藏

当用户角色是 `SHIPPER`：

```txt
隐藏：告警中心
保留：订单列表、订单详情、地址修改、智能问答、司机评分
```

即使客户手动访问 `/alerts`：

```txt
跳转 403 页面或订单列表
```

---

### 3.2 客户订单异常摘要

接口：

```http
GET /api/orders/{orderId}/exception-summary
```

展示位置：订单详情页顶部或物流状态卡片下方。

展示内容：

```txt
异常等级
异常摘要
处理状态
处理时间
```

---

## 4. 客户修改收货地址弹窗

### 4.1 页面入口

订单详情页按钮：

```txt
申请修改收货地址
```

按钮显示规则：

```txt
未签收显示
已签收隐藏或置灰
订单取消隐藏
```

---

### 4.2 表单字段

```txt
新收货地址
地图选点
联系人
联系电话
修改原因
```

---

### 4.3 地址输入提示

优先使用高德 JS API 输入提示；同时可调用后端代理：

```http
GET /api/amap/input-tips?keywords=xxx&city=杭州
```

用户选择地址后拿到：

```json
{
  "detail": "未来科技城海创园 3 号楼东门",
  "lng": 120.0165,
  "lat": 30.2798
}
```

---

### 4.4 改址影响预估

用户选完地址后自动调用：

```http
POST /api/orders/{orderId}/address-change-impact
```

展示卡片：

```txt
是否允许改址：允许/不允许
影响等级：微改址/近距离/中等影响/跨区/异常
额外距离：12.4km
预计延误：28分钟
额外费用：35元
是否需要调度审核：是
是否需要司机确认：是
原因说明：新地址距离原路线 2.1km，未超出配送范围
```

---

### 4.5 提交改址申请

```http
POST /api/orders/{orderId}/address-change-requests
```

成功后：

```txt
弹窗关闭
刷新改址历史
订单详情显示“改址申请审核中”
```

---

## 5. 改址历史进度

接口：

```http
GET /api/orders/{orderId}/address-change-history
```

展示成时间线：

```txt
客户提交申请
系统计算影响
调度员审核
司机确认
路线已更新
```

## 6. 调度员审核改址页面

可以放在：

```txt
调度管理 → 改址审核
或订单详情 → 改址申请卡片
```

列表字段：

```txt
申请时间
订单号
客户
原地址
新地址
影响等级
预计延误
额外费用
状态
操作
```

操作：

```txt
通过
拒绝
查看路线影响
```

接口：

```http
POST /api/address-change-requests/{requestId}/approve
POST /api/address-change-requests/{requestId}/reject
```
