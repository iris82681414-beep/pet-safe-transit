## 4. 需要对接的接口

### 4.1 路线规划

```http
POST /api/routes/plan
POST /api/routes/truck-plan
```

你要拿到：

```txt
distanceKm
durationMinutes
polyline
steps
restrictionWarnings
```

展示：

```txt
1. 地图上画路线
2. 卡片显示总距离、预计耗时、收费、限行提醒
3. steps 展示为路线说明
```

---

### 4.2 货物追踪和 ETA

```http
GET /api/cargo/{cargoId}/position
GET /api/cargo/{cargoId}/trajectory
GET /api/cargo/{cargoId}/eta
GET /api/orders/{orderId}/delay-prediction
GET /api/orders/{orderId}/risk-score
```

页面新增卡片：

```txt
预计到达：18:28
延误状态：可能延误
延误原因：改址导致路线增加 12.4km
运输风险：中风险 72 分
```

---

### 4.3 偏航纠偏

```http
POST /api/routes/deviation/check
POST /api/routes/reroute-suggestion
POST /api/vehicles/{plate}/command
```

页面交互：

```txt
1. 检测偏航后显示红色提示卡片
2. 展示偏航距离和持续时间
3. 显示纠偏路线
4. 按钮：一键下发纠偏路线
5. 下发后显示 command 状态
```

---

### 4.4 轨迹纠偏

```http
GET  /api/cargo/{cargoId}/trajectory/raw
GET  /api/cargo/{cargoId}/trajectory/corrected
POST /api/trajectory/correct
```

展示方式：

```txt
原始轨迹：灰色虚线
纠偏轨迹：蓝色实线
偏差较大点：红色点
```
