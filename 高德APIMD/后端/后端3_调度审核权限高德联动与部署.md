## 4. 调度员审核改址

### 4.1 审核通过

```http
POST /api/address-change-requests/{requestId}/approve
```

Request：

```json
{
  "remark": "新地址仍在配送范围内，允许改址",
  "notifyDriver": true,
  "recalculateRoute": true
}
```

处理逻辑：

```txt
1. 校验当前用户是 DISPATCHER 或 ADMIN
2. 查询 address_change_requests
3. 调用后端 2 /api/routes/replan 或内部服务重算路线
4. 更新订单目的地或卸货点
5. 更新改址申请状态为 APPROVED
6. 保存 address_change_logs
7. 如果 notifyDriver=true，生成调度指令 REROUTE
8. 通过 MQTT vehicle/{vin}/command 下发新路线
9. WebSocket 推送给前端
```

Response：

```json
{
  "code": 0,
  "data": {
    "requestId": "ADDR-20260701-001",
    "status": "APPROVED",
    "commandId": "CMD-20260701-001",
    "routeUpdated": true,
    "message": "改址已通过，新路线已下发司机"
  }
}
```

---

## 6. 文件上传接口

用于头像、车辆图片、卸货异常图片。

```http
POST /api/files/upload
Content-Type: multipart/form-data
```

FormData：

```txt
file: 图片文件
type: AVATAR / VEHICLE / UNLOAD_ABNORMAL / CARGO
```

Response：

```json
{
  "code": 0,
  "data": {
    "fileId": "FILE-20260701-001",
    "url": "https://your-domain.com/files/vehicle/xxx.jpg",
    "type": "VEHICLE",
    "size": 182736,
    "uploadedAt": "2026-07-01T10:00:00Z"
  }
}
```

实现建议：

```txt
1. 有 MinIO 就传 MinIO
2. 没 MinIO 就先传服务器本地 /uploads，并由 Nginx 静态访问
3. 数据库保存 file_id、url、type、created_by、created_at
```
