# 后端 1：订单地址、改址申请、司机评分与基础业务

> 你的定位：业务数据负责人。  
> 你不直接调用复杂路线规划，但你要把“客户改址、地址历史、司机评分、订单权限”这些业务数据做稳。

---

## 1. 你负责什么

你负责以下模块：

```txt
1. 客户收货地址修改申请
2. 地址修改记录和状态流转
3. 司机评分提交、查询、汇总
4. 订单基础权限校验
5. 订单、司机、车辆、客户之间的基础关联
6. 给后端 2 提供订单起点、终点、新地址、车辆当前位置等基础数据
7. 给前端 2 提供客户订单详情所需字段
```

## 3. 你要新增或修改的数据库表

### 3.1 address_change_requests

```sql
CREATE TABLE address_change_requests (
    id BIGSERIAL PRIMARY KEY,
    request_id VARCHAR(40) UNIQUE NOT NULL,
    order_id VARCHAR(40) NOT NULL,
    customer_id VARCHAR(40) NOT NULL,
    old_address TEXT NOT NULL,
    old_lng DOUBLE PRECISION,
    old_lat DOUBLE PRECISION,
    new_address TEXT NOT NULL,
    new_lng DOUBLE PRECISION NOT NULL,
    new_lat DOUBLE PRECISION NOT NULL,
    contact_name VARCHAR(50),
    contact_phone VARCHAR(30),
    reason TEXT,
    status VARCHAR(30) NOT NULL DEFAULT 'PENDING_REVIEW',
    impact_level VARCHAR(30),
    extra_distance_km NUMERIC(10,2),
    estimated_delay_minutes INT,
    extra_cost NUMERIC(10,2),
    need_dispatcher_review BOOLEAN DEFAULT TRUE,
    need_driver_confirm BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```

状态枚举：

```txt
PENDING_REVIEW    等待审核
AUTO_APPROVED     自动通过
APPROVED          调度员通过
REJECTED          调度员拒绝
DRIVER_CONFIRMED  司机确认
ROUTE_UPDATED     路线已更新
CANCELLED         客户取消
```

### 3.2 address_change_logs

```sql
CREATE TABLE address_change_logs (
    id BIGSERIAL PRIMARY KEY,
    request_id VARCHAR(40) NOT NULL,
    operator_id VARCHAR(40),

    action VARCHAR(50) NOT NULL,

日志 action 示例：

CALCULATE_IMPACT
AUTO_APPROVE
DISPATCHER_APPROVE
DISPATCHER_REJECT
DRIVER_CONFIRM
ROUTE_RECALCULATED
NOTIFY_DRIVER
```

### 3.3 driver_ratings

```sql
CREATE TABLE driver_ratings (
    id BIGSERIAL PRIMARY KEY,
    rating_id VARCHAR(40) UNIQUE NOT NULL,
    order_id VARCHAR(40) NOT NULL,
    customer_id VARCHAR(40) NOT NULL,
    driver_id VARCHAR(40) NOT NULL,
    plate VARCHAR(30),
    score INT NOT NULL CHECK (score BETWEEN 1 AND 5),
    punctuality INT CHECK (punctuality BETWEEN 1 AND 5),
    service_attitude INT CHECK (service_attitude BETWEEN 1 AND 5),
    cargo_integrity INT CHECK (cargo_integrity BETWEEN 1 AND 5),
    communication INT CHECK (communication BETWEEN 1 AND 5),
    comment TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(order_id, customer_id)
);
```

## 4. 你要提供的接口

### 4.1 提交改址申请

```http
POST /api/orders/{orderId}/address-change-requests
```

Request：

```json
{
  "newAddress": {
    "province": "浙江省",
    "city": "杭州市",
    "district": "余杭区",
    "detail": "未来科技城海创园 3 号楼东门",
    "lng": 120.0165,
    "lat": 30.2798
  },
  "contactName": "李先生",
  "contactPhone": "13800001111",
  "reason": "原收货地址临时无法卸货"
}
```

Response：

```json
{
  "code": 0,
  "data": {
    "requestId": "ADDR-20260701-001",
    "orderId": "SH-HZ-20260629-0291",
    "status": "PENDING_REVIEW",
    "impactLevel": "MEDIUM",
    "estimatedDelayMinutes": 28,
    "extraDistanceKm": 12.4,
    "extraCost": 35.00,
    "message": "改址会影响路线，需调度员审核"
  }
}
```

处理逻辑：

```txt
1. 校验当前用户是否为该订单客户
2. 校验订单状态是否允许改址
3. 调用后端 2 的 impact service 计算影响
4. 保存 address_change_requests
5. 保存 address_change_logs
6. 返回申请结果
```

### 4.3 客户提交司机评分

```http
POST /api/orders/{orderId}/driver-rating
```

Request：

```json
{
  "driverId": "DRV-001",
  "score": 5,
  "dimensions": {
    "punctuality": 5,
    "serviceAttitude": 5,
    "cargoIntegrity": 4,
    "communication": 5
  },
  "tags": ["准时送达", "沟通及时", "服务态度好"],
  "comment": "司机提前联系，卸货配合很好"
}
```

Response：

```json
{
  "code": 0,
  "data": {
    "ratingId": "RATE-20260701-001",
    "orderId": "SH-HZ-20260629-0291",
    "driverId": "DRV-001",
    "score": 5,
    "createdAt": "2026-07-01T10:20:00Z"
  }
}
```

校验规则：

```txt
1. 只有订单所属客户可以评价
2. 订单状态必须是 DELIVERED 或 SIGNED
3. 一个订单只能评价一次
4. score 必须 1~5
5. tags 最多 5 个
```
